# Власні значення і власні вектори (eigenvalues/eigenvectors)

## Що це

Матриця — це перетворення: береш вектор, `A @ v` зазвичай змінює і напрямок, і довжину. **Власний вектор** — виняток: вектор, для якого `A @ v` зберігає напрямок, тільки масштабує його. Формула: `A @ v = λ * v`, де `λ` (власне значення) — число-масштаб.

Це навмисно короткий і поверховий урок — глибина знадобиться пізніше, коли реалізовуватимемо PCA (Фаза 2).

## Для чого

PCA (Principal Component Analysis, головний метод зниження розмірності в класичному ML) — це буквально пошук eigenvectors коваріаційної матриці даних. Ці eigenvectors показують напрямки, вздовж яких дані найбільш "розтягнуті" (мають найбільшу варіацію) — саме ці напрямки і залишають, стискаючи розмірність.

## Розбір

### Інтуїція на прикладі

`S = [[2, 0], [0, 3]]` — розтягнення: по x у 2 рази, по y у 3 рази.

- `S @ [1, 0] = [2, 0]` — напрямок збережено, масштаб ×2 → власний вектор, λ=2
- `S @ [0, 1] = [0, 3]` — напрямок збережено, масштаб ×3 → власний вектор, λ=3
- `S @ [1, 1] = [2, 3]` — напрямок **змінився** (коефіцієнти масштабування різні по x і y) → не власний

Метафора: матриця як "прес" на гумовому листі — більшість напрямків повертає й викривлює, але є 1-2 "осі пресу", які тільки розтягуються.

### NumPy практика

```python
import numpy as np

A = np.array([[2, 1], [1, 2]])
eigenvalues, eigenvectors = np.linalg.eig(A)
print(eigenvalues)      # [3. 1.]
print(eigenvectors)     # стовпці — власні вектори!

v0 = eigenvectors[:, 0]        # ПЕРШИЙ вектор — стовпець 0, не рядок 0
lam0 = eigenvalues[0]
print(A @ v0)            # має бути ≈ lam0 * v0
print(lam0 * v0)
```

Для **симетричних** матриць (якою завжди є коваріаційна матриця) краще використовувати `np.linalg.eigh` — він гарантовано повертає дійсні числа, тоді як `eig` завжди повертає complex dtype (навіть якщо уявна частина 0).

### Спеціальні λ

- **`λ = 0`** — вектор "стискається у точку" (`A @ v = 0`). Матриця вироджена по цій осі.
- **`λ = 1`** — вектор взагалі не змінюється (`A @ v = v`). Тому одинична матриця `I` має **будь-який** вектор власним — усі λ=1.
- **`λ < 0`** — вектор масштабується і перевертається у протилежний бік.

### Звідки береться коваріаційна матриця (зв'язок з Уроком 02)

```python
cov = (X.T @ X) / (n - 1)
```

Це той самий `X.T @ X` з Уроку 02 — сума попарних dot product-ів колонок даних, тільки з нормуванням. Для 2 фіч (x, y):

```
cov = [[var(x),      cov(x,y)],
       [cov(x,y),    var(y)]]
```

`cov[0,0]` і `cov[1,1]` — дисперсія кожної фічі окремо, `cov[0,1]=cov[1,0]` — наскільки фічі "рухаються разом" (те саме поняття, що Pearson correlation з Фази 0).

### Інтерпретація для PCA

Якщо `np.linalg.eig(cov)` повертає `eigenvalues = [10.5, 0.02]` — це означає: вздовж першого eigenvector дані сильно розкидані (велика варіація), вздовж другого — майже не розкидані (це майже пряма лінія в цьому напрямку). **Другий eigenvector можна відкинути** без великої втрати інформації — саме так PCA стискає розмірність.

Приклад: дані у формі витягнутої "сигари" (сильна кореляція між x і y) мають один eigenvector з великим λ (вздовж довгої осі сигари) і один з крихітним λ (впоперек, де майже нема розкиду).

## Gotchas

- **`eigenvectors[:, i]`, не `eigenvectors[i]`** — `np.linalg.eig` повертає власні вектори **стовпцями**. Часте джерело багів.
- **`eig` завжди повертає complex dtype**, навіть коли уявна частина 0 (`3.+0.j`). Для симетричних матриць безпечніше `np.linalg.eigh`, яка одразу дає дійсні числа.
- **Eigenvalues існують тільки для квадратних матриць.** Для прямокутних — інший інструмент (SVD), не в цьому уроці.
- Комплексні eigenvalues трапляються для матриць обертання — поза межами цього уроку.
