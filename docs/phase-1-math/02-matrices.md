# Матриці як лінійні перетворення

## Що це

Матриця — це водночас дві речі: (1) **таблиця** — набір рядків/векторів однакової довжини (як дані: рядок = приклад, стовпець = фіча), і (2) **функція/перетворення** — щось, що бере вектор і повертає новий вектор. Обидва погляди правильні одночасно, просто застосовуються в різних контекстах.

## Для чого

Множення матриці на вектор (`A @ v`) — це те, що реально рахує кожен шар нейромережі (forward pass). А множення матриці на матрицю даних (`X @ q`) — це спосіб порахувати dot product / cosine similarity одразу для **всіх** рядків датасету, без циклу. Саме так vector-бази даних (Pinecone, pgvector, FAISS) шукають найсхожіші embedding серед мільйонів записів.

## Розбір

### Shape як контракт

Правило: `(m, k) @ (k, n) = (m, n)` — внутрішні розмірності мають збігатись, як типи функції в TS (не можна викликати `(A[]) => B` з `C[]`).

```python
(3, 4) @ (4, 1) = (3, 1)   # ok
(3, 4) @ (5, 4)            # помилка — 4 і 5 не збігаються
```

### Механіка множення: рядок × стовпець

`(A @ B)[i, j]` = dot product **рядка i з A** і **стовпця j з B**. Кожна клітинка результату — окремий dot product з Уроку 01.

```
A = [1  2]     B = [5  6]
    [3  4]         [7  8]

A @ B = [1*5+2*7   1*6+2*8]   = [19  22]
        [3*5+4*7   3*6+4*8]     [43  50]
```

### Три рівні "множення"

| Операція | Вхід | Вихід |
|---|---|---|
| `a @ b` (dot) | вектор, вектор | 1 число |
| `A @ v` | матриця, вектор | вектор (по числу на рядок A) |
| `A @ B` | матриця, матриця | матриця (сітка dot product-ів) |

### Некомутативність: `A @ B != B @ A`

Матриця — це перетворення, а `A @ B` — це **композиція** двох перетворень (спочатку `B`, потім `A`, як `compose(f,g)(x) = f(g(x))` в TS). Композиція функцій не комутативна, тому й множення матриць — теж:

```python
R = np.array([[0, -1], [1, 0]])   # обертання 90°
S = np.array([[2, 0], [0, 1]])    # розтягнення по x у 2 рази
v = np.array([1, 0])

R @ S @ v   # [0, 2] — спочатку розтягнули, потім повернули
S @ R @ v   # [0, 1] — спочатку повернули, потім розтягнули
```

Аналогія: одягнути шкарпетки → взуття ≠ взуття → шкарпетки. Порядок дій змінює результат.

### Транспонування, одинична, обернена матриця

- **`A.T`** — рядки стають стовпцями. `X.T @ X` — коваріаційна матриця (усі попарні dot product-и стовпців), використовується у PCA й лінійній регресії.
- **`np.eye(n)`** — одинична матриця. `I @ v = v` — як identity function `x => x`.
- **`np.linalg.inv(A)`** — обернена, "скасовує" `A` (`A @ A_inv ≈ I`, з дрібними float-похибками типу `8.88e-16` замість чистого 0). У проді ML її майже ніколи не рахують явно — замість `A_inv @ b` пишуть `np.linalg.solve(A, b)` (чисельно стабільніше).

### Векторизація пошуку найсхожішого (з Уроку 01)

```python
X = np.random.randn(100_000, 1536)         # 100k embedding
X = X / np.linalg.norm(X, axis=1, keepdims=True)
q = np.random.randn(1536); q /= np.linalg.norm(q)

scores = X @ q              # cosine similarity для всіх 100k одразу
top5 = np.argsort(scores)[-5:]   # топ-5 індексів (argsort сортує за зростанням)
```

### `A ** 3` vs `A @ A @ A`

`**` — поелементне піднесення в степінь (кожна клітинка окремо: `A[i,j] ** 3`). `@` — справжнє матричне множення (row·column dot product). Схожий синтаксис, зовсім різна математика — легко переплутати.

Для `A @ A @ A`: спочатку `AA = A @ A` (повна матриця), потім `AAA = AA @ A` — і тут важливо, що рядки для другого множення беруться вже з **накопиченого** `AA`, а не з оригінальної `A`.

## Gotchas

- **`*` vs `@`** знову: `A * B` (для матриць однакового shape) — поелементне множення (Hadamard), не матричне. Працює без помилки, але дає не той результат.
- **`A ** 2` замість `A @ A`** — поелементне піднесення, не квадрат матриці.
- **`np.random.rand([5, 5])` не працює** — `rand` приймає розміри як окремі аргументи (`rand(5, 5)`), не список. На відміну від `np.zeros([5,5])` чи `size=(5,5)` в інших функціях.
- **`np.linalg.norm(X)` без `axis`** рахує одне число на всю матрицю (Frobenius norm), а не по рядку. Для нормалізації embedding-рядків обов'язково `axis=1`.
- **`inv` числово нестабільна** — у реальному коді `np.linalg.solve(A, b)` замість `A_inv @ b`.
